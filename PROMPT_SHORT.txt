
1. Nhúng trang https://txahub.click toàn màn hình bằng iframe, nếu không được thì redirect bằng JavaScript.

Vui lòng mở app để sử dụng." Trên tooltip/thông báo này sẽ có link "Mở app" hoặc "Tải app", khi bấm vào sẽ hiện modal theo trình tự: Ảnh 1 (đang kiểm tra) -> nếu không phát hiện được app hoặc có lỗi -> Ảnh 2 (hướng dẫn cài app). URL để mở app là txahub://.

3. Ở đầu trang có thông báo về các tính năng chỉ dành riêng cho app Android, chưa hỗ trợ Windows.

5. Các tính năng sẵn có của trang txahub.click thuộc admin (PRO, TXAPRO...) cũng bị chặn tương tự, tooltip và logic giống như trang admin view khi thiếu giấy phép TXAPRO. Một vài tính năng có sẵn trên website sẽ không hoạt động trên app (chỉ hoạt động trên website), cũng áp dụng logic chặn tương tự: làm mờ, disabled, có tooltip và thông báo trên đầu trang giống logic TXAPRO. Tooltip/thông báo cũng có link để mở app hoặc tải app, khi bấm vào hiện modal theo trình tự: Ảnh 1 (đang kiểm tra) -> nếu không phát hiện được app hoặc có lỗi -> Ảnh 2 (hướng dẫn cài app).

6. Nhận diện Android app qua user agent: User agent của app Android sẽ là chuỗi hoàn toàn mới có định dạng TXAAPP_[Tên_Thiết_Bị]_Android_[Phiên_Bản] (ví dụ: TXAAPP_Samsung_Galaxy_S21_Android_12). Không phải là user agent gốc + TXAAPP_ mà là chuỗi hoàn toàn mới chỉ chứa TXAAPP_... Code để nhận diện: navigator.userAgent.includes('TXAAPP_') hoặc navigator.userAgent.startsWith('TXAAPP_'). Nếu có chuỗi này thì unlock các tính năng, nếu không thì lock như trên.

7. Tạo file readme_app.md mô tả chi tiết cấu trúc/thành phần/logic nhận diện/chặn-mở chức năng, và hướng dẫn cho developer Android: App Android phải set user agent WebView là chuỗi hoàn toàn mới với định dạng TXAAPP_(tên thiết bị + hệ điều hành), KHÔNG phải user agent gốc + TXAAPP_. Ví dụ: TXAAPP_Samsung_Galaxy_S21_Android_12. Code Android: val deviceName = Build.MODEL.replace(" ", "_"); val osVersion = "Android_${Build.VERSION.RELEASE}"; webView.getSettings().setUserAgentString("TXAAPP_${deviceName}_${osVersion}"); (Lưu ý: KHÔNG cộng với user agent gốc, chỉ set chuỗi mới hoàn toàn).

8. Yêu cầu: Sử dụng vanilla JavaScript, CSS responsive mobile-first, modal có animation, tooltip có animation, code có comment tiếng Việt.

9. Thông tin: Website txahub.click, deep link txahub://, ảnh 1 và 2 dùng placeholder div với text tạm thời.

9.1. Xử lý Deep Link txahub:// (QUAN TRỌNG): App Android tự động chuyển đổi txahub://path thành https://txahub.click/path. Ví dụ: txahub://dashboard -> https://txahub.click/dashboard, txahub://verify-mail/abc123 -> https://txahub.click/verify-mail/abc123. Website không cần xử lý đặc biệt, chỉ cần đảm bảo routes hoạt động bình thường. Nếu website có link txahub:// trong nội dung, có thể giữ nguyên (app sẽ xử lý) hoặc chuyển đổi thành URL web khi hiển thị trên trình duyệt.

9.2. Xử lý link xác minh email (QUAN TRỌNG): Khi người dùng bấm vào link xác minh email (ví dụ: https://txahub.click/verify-mail/abc123), website tự động detect và redirect đến deep link tương ứng (txahub://verify-mail/abc123) để mở app. Không hiển thị trên website, tự động mở app nếu đã cài. Detect các routes: /verify-mail/, /verify-email/, /confirm/, /verify/. Code JavaScript tự động chuyển đổi URL web thành deep link và thử mở app. Nếu app chưa cài, có thể hiển thị modal hướng dẫn cài app sau 2 giây.
